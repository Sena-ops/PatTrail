Note: This is a minimal viable product (MVP) API for creating and listing pets. Duplicate pet names are permitted in this MVP as explicitly documented.

Base URL

http://localhost:8080

Authentication

No authentication is required for this MVP.

Content Type

All requests and responses use application/json content type.

Endpoints

GET List All Pets

GET /api/pets

Returns a list of all pets with their id, name, and species.

Response

200 OK
[
  {
    "id": 1,
    "name": "Rex",
    "species": "CACHORRO",
    "createdAt": "2025-08-13T20:28:37.517"
  },
  {
    "id": 2,
    "name": "Mimi",
    "species": "GATO",
    "createdAt": "2025-08-13T20:29:04.231"
  }
]

POST Create Pet

POST /api/pets

Creates a new pet with the provided name and species.

Request Body

{
  "name": "Rex",
  "species": "CACHORRO"
}

Validation Rules

Field Type Required Constraints Allowed Values
name string Yes 1-60 characters, not blank Any string
species string Yes Must be one of the allowed values CACHORRO, GATO

Success Response

201 Created
{
  "id": 1,
  "name": "Rex",
  "species": "CACHORRO",
  "createdAt": "2025-08-13T20:28:37.517"
}

Error Response

400 Bad Request
{
  "code": "VALIDATION_ERROR",
  "message": "One or more validation errors occurred.",
  "details": [
    {
      "field": "name",
      "issue": "must not be blank"
    }
  ]
}

Error Response Format

All validation errors follow this standard JSON format:

{
  "code": "VALIDATION_ERROR",
  "message": "One or more validation errors occurred.",
  "details": [
    {
      "field": "field_name",
      "issue": "description of the validation issue"
    }
  ]
}

Examples

Valid Pet Creation

Request: 
curl -i -X POST http://localhost:8080/api/pets \
  -H "Content-Type: application/json" \
  -d '{ "name": "Rex", "species": "CACHORRO" }'

Another Valid Pet Creation

Request: 
curl -i -X POST http://localhost:8080/api/pets \
  -H "Content-Type: application/json" \
  -d '{ "name": "Mimi", "species": "GATO" }'

Invalid Species (Error Example)

Request: 
curl -i -X POST http://localhost:8080/api/pets \
  -H "Content-Type: application/json" \
  -d '{ "name": "Piu", "species": "PASSARO" }'

List All Pets

Request: 
curl -i -X GET http://localhost:8080/api/pets
